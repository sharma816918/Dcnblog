Understanding Dataflow in DCN

In any data communication system, *dataflow* defines the direction in which information travels between devices. Based on this directionality, there are three primary transmission modes: Simplex, Half Duplex, and Full Duplex. Each mode has distinct characteristics, advantages, and real-world applications.

1. Simplex Mode

In Simplex mode, data travels only in one direction ‚Äî from the sender to the receiver. The receiver cannot send any data back. Examples include a monitor receiving data from a CPU or a keyboard sending data to the CPU.

2. Half Duplex Mode

In Half Duplex mode, data transmission can occur in both directions, but not at the same time. One device must wait until the other finishes sending data before responding. Classic examples include walkie-talkies and CB radios.

3. Full Duplex Mode

Full Duplex mode allows data to be sent and received simultaneously between devices. This bidirectional communication improves efficiency and speed, making it ideal for telecommunication and modern Ethernet networks.
